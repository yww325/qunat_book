### 关键产出

**第1章核心框架（可脱书背诵版）：**

1. **为什么重要**：数据丰富信息贫乏 → 数据坟墓需要变成知识金块
2. **是什么**：KDD七步流程中的核心步骤；从RDB/仓库/事务/高级DB/WWW挖掘有趣知识
3. **能挖什么数据**：关系库/仓库/事务库/对象-关系/空间/时序/文本/多媒体/WWW
4. **能挖什么模式**：特征化/区分（概念描述）+ 关联 + 分类/预测 + 聚类 + 演变 + 偏差分析
5. **什么叫有趣**：客观（支持度+置信度+新颖+有效+有用+可理解）× 主观（出乎意料+可行动+验证假设）
6. **三大挑战**：
   - 技术/UI：多粒度交互挖掘 + 背景知识 + 专用查询语言 + 可视化 + 噪音处理 + 主观兴趣度
   - 性能：可规模化（线性增长）+ 并行/分布/增量算法
   - 数据多样性：复杂类型 + 异种/Web挖掘

**支持度/置信度公式（关联规则核心）：**
```
supp(X⇒Y) = P(X∪Y)   # 满足规则样本占比
conf(X⇒Y) = P(Y|X)    # 含X也含Y的条件概率
```

**KDD七步**：清理→集成→选择→变换→挖掘→评估→表示
