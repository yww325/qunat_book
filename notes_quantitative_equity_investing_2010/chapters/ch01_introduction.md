# 第1章 引言（Introduction）

## 1. 这一章解决什么问题？

- 给出“量化权益投资”在投资管理中的定位：它不是某个单一模型，而是一套**系统化、可复制、可审计**的决策流程。
- 解释为什么需要量化：数据规模、计算能力、风险管理与监管要求，使得“凭直觉做组合”越来越难以稳定。
- 对市场有效性给出更现实的视角：市场并非静态有效，策略会随参与者学习而衰减，需要持续验证与迭代。

## 2. 核心观点（用自己的话复述）

- **量化不是“更聪明”，而是“更一致”**：把投资过程规则化，减少情绪与不一致的判断。
- **风险量化是核心工程**：没有风险模型与约束系统，所谓“alpha”大概率是隐含风险暴露。
- **市场不是一次性解决的问题**：策略要面对结构变化、制度变化、参与者变化。

## 3. 方法论框架（这一章给的“总流程”）

把量化权益投资看成流水线：

1) 明确投资目标与约束（基准、风险预算、合规、容量）
2) 获取并治理数据（质量、对齐、滞后、异常）
3) 建立可解释的预测信号（因子/模型）
4) 建立风险模型（因子风险/特异风险/尾部风险）
5) 组合构建（优化 + 约束 + 稳健化）
6) 交易执行（成本、冲击、拆单、算法）
7) 监控与归因（收益归因、风险归因、漂移检测）
8) 迭代（样本外检验、上线/下线机制）

## 4. 易错点 / 容易被忽略的坑

- 把“量化”误解成“机器学习/复杂模型”：作者更强调**流程纪律 + 风险与成本约束**。
- 忽视“可执行性”：没有交易成本与容量约束的回测，结论往往不可用。
- 把市场当作静态：策略有效性会随市场结构变化而变化。

## 5. 和A股/港股落地的连接

- A股：
  - 更要强调交易约束（涨跌停、停牌、ST）与成本/冲击建模；否则“纸面alpha”会塌。
  - 市场风格切换快，策略漂移监控（比如IC崩溃报警）要更敏感。
- 港股：
  - 个股流动性与价差分化大；组合权重上限、最小成交额与分批执行更关键。
