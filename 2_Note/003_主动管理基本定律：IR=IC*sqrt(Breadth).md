# 003_主动管理基本定律：IR=IC*sqrt(Breadth)

> 来源：《主动投资组合管理》（第二版）PDF 第 61–110 页（本段主要是风险模型/主动收益与附加值/信息率引入，为“主动管理基本定律”做铺垫）。

## 1) 本段覆盖的章节/主题（按可识别内容）

- 风险（Risk）与结构化风险模型
  - 多因子风险模型的必要性：把共同因子与特异性风险分开

- 超额收益、业绩基准与附加值（Value Added）

  - IR 作为描述主动管理“机会集/能力-风险比”的核心指标

## 2) 主动管理基本定律：定律本身 + 有效 Breadth 的折损

### 2.1 定律（后续章节会严格推导，但这里先给“可执行版本”）

```text
IR ≈ IC * sqrt(Breadth)
```

- `IR`：信息比率（Information Ratio）
- `IC`：信息系数（Information Coefficient），预测与实现收益的相关性
- `Breadth`：独立决策次数（不是“交易次数”）

**前提假设（必须知道，否则会在实盘踩坑）：**
- 决策是“近似独立”的（独立性不足会严重高估 Breadth）；
- 预测误差是可控且可估计的；
- 交易成本/冲击成本没有把信号完全吃掉（否则 IR 会在落地时塌陷）。

### 2.2 相关性如何折损有效 Breadth（Effective Breadth）

你可以把 “Breadth” 先做成一个可操作的近似：

```text
Breadth_eff = N / (1 + (N-1)*rho)
```

- `N`：表面上的“决策数”（例如同时覆盖的标的数/触发次数）
- `rho`：决策之间的平均相关性（信号相关/标的相关/同一行情段聚簇）

直觉：
- 当 `rho → 0`：`Breadth_eff ≈ N`（独立性好）
- 当 `rho → 1`：`Breadth_eff → 1`（你以为很多机会，实际上只有一个同质机会）

这也是本段强调结构化风险模型的原因：
- 风险模型把“共同因子驱动的相关性”显式化，才能识别真正的特异性机会。

## 3) IC 的估计与衰减（如何避免样本内幻觉）

### 3.1 IC 的实务估计：别只看“命中率”
IC 本质是相关性，不是胜率：
- 同样 60% 胜率，若盈亏比不稳定/尾部风险大，IC 可能很差；
- 同样低胜率，若收益分布偏正、尾部可控，IC 反而可能可用。

建议你在事件级别记录并输出：
- `fwd_ret_15m`、`fwd_ret_30m`、`fwd_ret_60m`、`fwd_ret_eod`
- `score`（信号强度，标准化后更好）

然后分别计算：
- `IC_15m`、`IC_30m`、`IC_60m`、`IC_eod`

### 3.2 IC 衰减（IC Decay）
高频信号最常见的失败模式是“有效期变短”：
- 信号在 15m 有效，但到 60m/eod 失效
- 或者在过去有效、近期快速衰减

落地方式：
- 输出一条 `IC_decay_curve = [IC_15m, IC_30m, IC_60m, IC_eod]`
- 并用滚动窗口跟踪（例如最近 60 次触发）。

### 3.3 如何避免样本内幻觉
- 把基准解释掉：不要把大盘/板块上涨当作你的预测能力
- 只在样本外评价：walk-forward 的 test split 才是“能力更真实”的估计
- 置信区间：触发次数少时，稳定率/强反弹率的统计波动巨大

## 4) 3 个常见误区/陷阱（偏实操）

### 误区 1：把“总风险”当成“主动风险”
表现：策略回测波动不大，但其实只是跟着大盘 β 在走。

避免：
- 把 `market_ret` 与 `residual_ret` 固化为回测字段；
- 评价指标优先看相对基准的稳定性（例如残差 stable_rate）。

### 误区 2：因子暴露被低估（尤其在极端区间）
表现：某个风格/行业暴露在极端行情时呈现非线性风险（流动性枯竭/点差放大）。

避免：
- 在你的 15m 扫描/回测里，对“低成交额时段”的触发进行单独分组统计；
- 引入 `min_dollar_volume`（你已做）并把通过率写进报表。

### 误区 3：把“运气”当“能力”（短样本错觉）
表现：某几次触发收益很好，就认定参数最优。

避免：
- walk-forward 必须看多 split、一致性；
- 目标函数加稳定性项（你已有 stable/strong 加权思路），再加 `effective_trig`（见下）避免聚簇样本骗分。

## 5) 对港股 15m 项目的 2 条落地改造（字段/阈值/实验）

### 改造 1：有效 Breadth（去聚簇）指标落地：effective_trig + cluster_ratio

新增字段：
- `effective_trig`：冷却窗口内重复触发只算 1 次（例如 `cooldown_bars=8`，=2小时）
- `cluster_ratio = effective_trig / trig`

建议阈值（先经验，后用历史校准）：
- 若 `cluster_ratio < 0.6`：说明触发高度聚簇，容易过拟合单一行情段；
  - 在 grid search / walk-forward objective 里扣分或直接剔除。

实验：
- A：现有 trig_floor
- B：用 effective_trig_floor（例如 `effective_trig >= 4`）
对比 test split 下的稳定率变化。

### 改造 2：IC_decay 曲线 + walk-forward 自动停用（degrade rule）

新增字段（每个 ticker/参数在每个 split 的 test 部分输出）：
- `IC_15m_test`, `IC_60m_test`, `IC_eod_test`
- `IC_decay_slope`（例如从 15m→eod 的线性斜率）

停用规则（可写成策略层风控）：
- 若连续 2 个 split：
  - `IC_eod_test < 0.03` 或 `IC_decay_slope` 明显为负（例如 `< -0.02`），则标记 `degraded=true`，暂停该 ticker 参数并重新网格搜索。
