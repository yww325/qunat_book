## 2) 朴素预测（naive forecast）vs 精炼预测（refined forecast）：区别与组合中的作用

### 2.1 朴素预测：没有信息时的“无条件期望”
原文用 Charlie 的故事点出：
- 即便一个人“2/3 时间说对市场上涨”，也可能**没有价值**，因为市场在长期/统计意义上本来就更偏向上涨。
- 朴素预测对应的是：**在没有任何额外信息时，我们对收益的无条件期望**（例如月度 0.5%）。

朴素预测在组合里扮演的角色：
- 它是 baseline（基线），把你拉回“不要乱动”的状态；
- 精炼预测的价值，来自于你能在这个 baseline 之上提供可验证的增量信息。

### 2.2 精炼预测：把 Raw 信号标准化后，用 IC 和波动率把它变成收益量级
原文给出一套非常“工程化”的流程：

- 定义涨跌二元预测：

```text
Raw(t)=+1 表示预测上涨；Raw(t)=-1 表示预测下跌
```

- 如果预测分布与历史上涨概率一致（例如 2/3 的 Raw=+1），则 Raw 的：
  - 均值 = 1/3
  - 标准差 = 0.9428

- 将 Raw 标准化后得到 Score（z 值），原文给出：
  - 上涨时 Score ≈ 0.707
  - 下跌时 Score ≈ -1.414

- 精炼预测的核心形式（“无条件均值 + 波动率×IC×Score”）：

```text
Refined Forecast = μ + σ * IC * Score
```

原文例子里：
- 月度无条件均值 μ = 0.50%
- 月度标准差 σ = 5.20%
- 因此：
  - 预测上涨：`0.50 + 5.20*IC*0.707`
  - 预测下跌：`0.50 - 5.20*IC*1.414`

这一步的要点：
- **同一个“涨/跌观点”在不同波动环境下应该输出不同量级的 alpha**（σ 缩放）；
- 预测能力不是“你说得多响”，而是 `IC`（相关性）这一件事。

## 3) 信号校准：score 标准化、IC 估计、波动率缩放（含关键公式）

### 3.1 score 标准化：把分布变成“零均值/单位方差”
你如果不把 Raw/评级/排序变成标准分，后续的组合或筛选会被量纲污染：
- 同样是“买入”，对低波动标的和高波动标的贡献不应相同。

### 3.2 (10-30) 预测对波动率预测的影响（只取决于 IC，且很小）
原文给出（10-30）：

```text
σ_后验 = σ_先验 * sqrt(1 - IC^2)    (10-30)
```

关键结论（原文反复强调）：
- 收益预测对波动率预测的影响**极其微小**（因为现实 IC 通常很小，例如 0–0.15）；
- 更重要的是：它**只与 IC 有关**，与“你预测涨多少/跌多少”无关。

### 3.3 (10-31) 预测对相关性预测的修正（同样主要由 IC 决定）
原文在两资产/两预测例子里给出（10-31）：

```text
ρ* = ρ * [ (1 - IC_M * IC_L) / sqrt((1-IC_M^2)(1-IC_L^2)) ]    (10-31)
```

含义：
- 你可能“预测标普差、中盘好”，但这属于**条件均值**的变化；
- 不应因此冲动地把历史相关系数 0.95 改成负相关。
- 相关性预测的修正很小，而且只由预测能力（IC）决定。

## 4) 3 个实操误区（含随机输入测试）

### 误区 1：把条件均值当成条件方差
- 看好/看空属于均值预测，不等于风险结构就变了。
- 解决：收益预测（alpha）和风险预测（协方差/波动率）分开处理。

### 误区 2：滥改相关系数
- 看到“观点相反”就想把相关性从 0.95 改成负，是典型错误。
- 解决：不要用观点去改协方差矩阵；除非有独立证据证明结构变更。

### 误区 3：复杂模型失控（你被技术控制，而不是你控制技术）
原文给出的指导原则非常实用：
- 从简单模型过渡到复杂模型时，必须先掌握简单情形，理解缺陷再升级。

随机输入测试（sanity check）：
- **输入完全随机的数据**（白噪声/打乱序列），观察复杂模型输出。
- 若随机数据也能“预测成功”，说明存在严重问题（过拟合/泄露/实现 bug）。

## 5) 对港股 15m 项目的 2 条落地（字段/阈值/实验）

### 落地 1：把“信号→收益量级”的校准做成标准流程（score_z + IC_prior + vol scaling）
新增字段：
- `score_z`：把触发信号（dd/J/RSI/量等）标准化（零均值/单位方差）
- `IC_prior_15m`：滚动估计的先验 IC（建议 EWMA 更新）
- `sigma_15m`：15m 滚动波动率

定义 refined_alpha：

```text
refined_alpha_15m = sigma_15m * IC_prior_15m * score_z
```

阈值（先保守）：
- 若 `IC_prior_15m < 0.02`：将 refined_alpha 打折（例如 ×0.5）或直接降低推送优先级。

实验：
- A：仅用触发条件
- B：触发条件 + refined_alpha 门槛（例如 top 20% 才推）
对比：触发数量、stable/strong、以及样本外表现。

### 落地 2：风险矩阵稳定性规则 + 随机输入 sanity check
- 规则 1：**禁止因短期观点修改相关系数/协方差矩阵**。
  - 仅允许用长期窗口估计的协方差（例如 20d/60d），并按固定频率更新。
- 规则 2：加入 `random_input_check`：
  - 每次发布新版本或更换参数/score 组合时：
    - 打乱收益序列或用白噪声替代
    - 若策略仍出现显著“胜率/收益”，判定为泄露或实现错误，阻止上线

walk-forward：
- 将 sanity check 与 walk-forward 结合：训练期选参数，测试期验证；并在测试期额外做随机对照组。
