## 2) 信息分析要解决的问题：如何估计信息价值 / IC / 半衰期

信息分析的目标不是“讲故事”，而是回答三件工程问题：

1) 这个信息源有没有预测力？（IC 是否稳定为正）
2) 预测力能持续多久？（IC_decay / half-life）
3) 在成本与约束下，信息还能剩多少净价值？（净 edge / VA 代理）

本段的关键思想：
- **IC 是“质量”指标**，IR/VA 是“价值”指标；
- half-life 决定了你该不该、以及多久调仓一次。

一种常见的衰减表达（以指数衰减为例）：

```text
IC(j) = IC(1) * γ^(j-1)
```

- `j`：未来第 j 个周期（例如第 j 根 15m bar）
- `γ`：衰减因子（0~1）

## 3) 事件研究（event study）：核心步骤与注意事项

### 3.1 核心步骤（最小可行版）
1) 定义事件与事件时间（event time）：
   - 例如财报公告、评级变更、政策新闻
2) 定义事件窗口（event window）：
   - 例如 [t0, t0+4]（未来 1 小时的 15m bar）或到 EOD
3) 计算事件后的“残差收益”（剔除市场/行业）：
   - 避免把 β/板块行情当成事件 alpha
4) 聚合统计：
   - 平均残差收益、显著性、以及随时间的衰减曲线

### 3.2 两个关键定义/公式（用于落地）
（1）事件后累计残差收益（概念）：

```text
θ(1,j) = residual_return accumulated from 1 to j
```

（2）预测必须与风险量级匹配（“精炼预测”的事件版）：

```text
α(1,j) = ω(1,j) * [ b0 + Σ_k X_k * b_k ]
```

含义：
- 事件解释变量（X_k）需要通过系数 b_k 形成“预测强度”，
- 再用波动率 ω 把它换算到收益量级。

注意事项（高频尤其重要）：
- 时间对齐：事件发生时间与 15m bar 的边界对齐，否则会出现“提前知道”的假象。
- 口径一致：事件窗口收益要用同一基准/残差口径，不要混 absolute 与 residual。

## 4) 3 个实操误区（必须防）

### 误区 1：数据挖掘 / 多重检验（p-hacking）
- 你试 20 个无信息的因子，95% 置信水平下“至少一个看起来显著”的概率会很高。

对策：
- random baseline / permutation test（见第5节）。

### 误区 2：幸存者偏差（survivorship bias）
- 只看“还活着”的标的/基金，会系统性高估效果。

对策：
- 对股票池变更做记录；回测时固定当期可交易域。

### 误区 3：时间对齐错误（look-ahead）
- 高频里最常见：用到了“本 bar 收盘后的信息”。

对策：
- 每条信号记录 `signal_ts` 与 `last_data_ts`，强制 `last_data_ts < signal_ts`。

## 5) 对港股 15m 项目的 2 条落地（字段/阈值/实验）

### 落地 1：信号衰减审计（IC_decay / half-life）+ 动态阈值/调仓

新增字段（事件级别）：
- `fwd_ret_15m`, `fwd_ret_30m`, `fwd_ret_60m`, `fwd_ret_eod`
- `IC_15m`, `IC_30m`, `IC_60m`, `IC_eod`（滚动估计）
- `half_life_bars`

规则（示例）：
- 若 `half_life_bars <= 2`（≈30分钟）：
  - 提高推送门槛（例如 `score_z` 阈值 +0.5）
  - 或延长 cooldown，避免在信息已衰减时频繁触发。

实验：
- A：固定阈值
- B：half-life 驱动动态阈值
对比：净触发数、净 edge（扣成本）与样本外稳定性。

### 落地 2：反 p-hacking：random baseline + permutation test

新增实验（每个新因子/新规则必须过门槛）：
- `random_baseline`：保持触发频率相近的随机策略
- `permutation_test`：在同一时间戳内随机置换因子值，重新计算 IC/t-stat 分布

准入门槛（先保守）：
- 只有当真实因子的 `IC`/`objective` 位于 permutation 分布的 **95% 分位以上**，才允许进入 refined_alpha 池。
